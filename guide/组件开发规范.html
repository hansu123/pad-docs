    # 组件编写规范

    # 一. 页面组件开发的编写顺序都统一风格

    • name
    • mixin
    • props (可以按开头字母顺序排列)
    • data
    • computed
    • components
    • watch
    • methods
    • 生命周期

    # 二. 编写组件的结构

     ├── check
     ├── subpage    //子页面
         ├── xx.js     //子页面1
         ├── xx.js     //子页面2
     ├── components     //页面组件
         ├── xx.js     //页面组件1
         ├── xx.js     //页面组件2
     ├── index.js     //主文件入口文件
     └── checkDetail.js     //主文件

    请务必不要直接使用index.js作为组件文件名，因为在编辑器中不好查找

    # 三. 编写通用组件的分类

    • 通用组件放在components/common中（一般是ui组件）

    • 通用内容组件放在components/content中 （与这个项目有关而且经常会用到的组件）
