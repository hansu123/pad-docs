    # 样式开发规范

    # 一. 样式结构目录

    ├─ styles
│  ├─ components    //存放所有ui组件的样式
│  │  └─ XXX     
│  ├─ base.scss      //基础样式配置
│  ├─ common.scss    //和页面相关的样式
│  ├─ func.scss      //全局常用样式类配置
│  ├─ global.scss    //全局scss变量配置
│  ├─ mixin.scss     //全局mixin配置
│  └─ index.scss     //样式入口文件

    index.scss

    @import "./base.scss";
@import "./common.scss";
@import "./func.scss";
@import "./mixin.scss";
@import "./global.scss";
@import "./components/index.scss";

    在main.js中引入

    import "@/styles/index.scss"

    # 二. 开发样式规范

    • 通用组件的样式请放在styles/components下

    • 项目有关组件内部的样式使用mixin.scss,global.scss等样式需要单独引入

    • 项目有关组件内部的样式可以放在组件内部也可以单独抽离样式文件

    • 严格遵守sass使用规范
